# 🎨 메인 페이지 (/) UI 명세서 - Updated

이 문서는 Smart Portfolio의 메인 랜딩 페이지(/)에 대한 상세 UI 명세입니다.

## 1. 레이아웃 구조 (Layout Structure)
- **상단 (Header)**: 네비게이션 바 (로고 + 메뉴 링크)
- **섹션 1: 히어로 (Hero)**: 강렬한 자기소개 + 프로필 이미지 + 메인 CTA 버튼
- **섹션 2: 스킬 (Skills)**: 보유 기술 스택 태그/뱃지 나열
- **섹션 3: 경력 (Career)**: 주요 이력 및 경험 타임라인
- **섹션 4: 프로젝트 목록 (Projects)**: **[핵심]** 카테고리 필터 + 프로젝트 카드 그리드
- **공통 컴포넌트**: **AI 챗봇 위젯** (우측 하단 플로팅 고정)
- **하단 (Footer)**: 연락처, SNS 링크, 저작권, **Admin 로그인 링크**

---

## 2. 각 섹션별 상세 (Section Details)

### 2.1 상단 네비게이션 (Header)
- **로고**: 좌측 정렬. 텍스트 로고 ("Smart Portfolio" 또는 사용자 이름).
- **메뉴**: 우측 정렬. `About`, `Works`, `Contact` 링크.
- **동작**: 스크롤 시 상단 고정 (Sticky) 또는 배경색 투명 -> 불투명 전환.

### 2.2 히어로 섹션 (Hero)
- **구성**:
    - **타이틀**: "비즈니스 문제를 기술로 해결하는 개발자, 김코드입니다." (H1)
    - **서브텍스트**: "단순한 코딩을 넘어 가치를 만듭니다. 제 AI 비서에게 무엇이든 물어보세요." (P)
    - **CTA 버튼 (Primary)**: "프로젝트 보러가기" → 클릭 시 **섹션 4 (Projects)**로 스무스 스크롤.
    - **CTA 버튼 (Secondary)**: "AI에게 질문하기" → 클릭 시 우측 하단 챗봇 위젯 열림.
- **프로필 이미지**: 우측(데스크톱 기준) 배치. 300x300px 이상의 고화질 원형 또는 라운드 사각형 이미지.

### 2.3 스킬 섹션 (Skills)
- **구성**:
    - **타이틀**: "Tech Stack"
    - **스킬 그룹**: 프론트엔드 / 백엔드 / 도구 등으로 구분 권장.
    - **태그 디자인**: 둥근 모서리(Pill shape) 뱃지. 기술별 브랜드 컬러 배경색 적용 (예: React-파랑, Node-초록).
- **인터랙션**: 마우스 호버 시 살짝 확대(Scale up).

### 2.4 경력 섹션 (Career)
- **구성**:
    - **타이틀**: "Work Experience"
    - **타임라인 아이템**:
        - **기간**: `2023.01 - 현재` (강조 텍스트)
        - **조직명**: 회사 또는 팀 이름.
        - **역할**: `Senior Frontend Developer`
        - **설명**: 주요 성과 1~2줄 요약 (Bullet points).

### 2.5 ★ 프로젝트 목록 (Projects)
- **구성**:
    - **타이틀**: "Representative Projects"
    - **카테고리 필터 (Filter)**: 상단 중앙 배치. `전체` | `Web` | `App` | `Design` 등 (Pill 형태 버튼). 선택 시 해당 카테고리 아이템만 필터링 (Animation 적용).
    - **그리드 (Grid System)**:
        - **Desktop**: 3열 (3 Columns)
        - **Tablet**: 2열 (2 Columns)
        - **Mobile**: 1열 (1 Column)
- **카드 디자인 (Card Item)**:
    - **썸네일**: 상단 배치. 16:9 또는 4:3 비율. `object-cover`.
    - **정보 영역**: 썸네일 하단에 배치.
        - **카테고리 태그**: 작고 은은한 색상의 뱃지 (예: `Web Development`).
        - **프로젝트 제목**: 굵은 폰트 (H3), 1~2줄 제한.
        - **간략 설명**: 프로젝트의 핵심 가치를 담은 2줄 요약 (Text-truncate).
- **인터랙션 (Interaction)**:
    - **Hover**: 카드 전체가 살짝 위로 떠오름 (Translate Y -5px) + 그림자 강화 (Box-shadow-lg) + 썸네일 이미지 살짝 확대 (Zoom-in).
    - **Click**: 해당 프로젝트의 **`/project/[id]` (상세 페이지)**로 이동.

### 2.6 AI 챗봇 위젯 (Floating Chatbot)
- **위치**: 화면 우측 하단 고정 (z-index 최상위).
- **초기 상태 (Collapsed)**: 챗봇 아이콘(🤖) + "무엇이든 물어보세요" 말풍선.
- **열린 상태 (Expanded)**: 채팅 인터페이스 (헤더, 메시지 목록, 입력창).
- **특징**: 스크롤이 프로젝트 섹션에 도달하면 "이 프로젝트들에 대해 설명해 드릴까요?"라고 토스트 메시지를 먼저 띄움 (Active Sales).

### 2.7 하단 (Footer)
- **구성**:
    - 이메일, GitHub/LinkedIn 아이콘 링크.
    - Copyright 텍스트.
    - **Admin Login**: 맨 하단 구석에 작고 흐릿하게 배치 (일반 사용자 눈에 띄지 않게).

## 3. 반응형 동작 (Responsive Behavior)

### 📱 모바일 (Mobile, < 768px)
- **네비게이션**: 로고만 표시하고 메뉴는 '햄버거 버튼'으로 숨김. 클릭 시 드로어(Drawer) 메뉴 오픈.
- **히어로**: 세로 배치 (텍스트 위, 이미지 아래). 텍스트 중앙 정렬.
- **프로젝트**: **1열(1 Column)** 카드 리스트. 썸네일을 크게 보여줌. 필터 버튼은 가로 스크롤(Horizontal Scroll) 처리.
- **챗봇**:
    - 아이콘은 우측 하단 유지.
    - 열릴 때 화면 전체를 덮는 **풀스크린(Full Screen)** 모달 또는 높이 80%의 **바텀 시트(Bottom Sheet)**.

### 💻 데스크톱 (Desktop, > 1024px)
- **히어로**: 좌우 분할 배치 (텍스트 왼쪽, 이미지 오른쪽). 여백(Whitespace) 충분히 활용.
- **프로젝트**: **3열(3 Columns)** 그리드.
- **네비게이션**: 상단 메뉴 링크 나열.
- **챗봇**: 우측 하단에 400x600px 정도의 팝업 윈도우 형태.
