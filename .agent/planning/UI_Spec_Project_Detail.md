# 🎨 프로젝트 상세 페이지 (/project/[id]) UI 명세서

이 문서는 Smart Portfolio의 개별 프로젝트 상세 페이지(`/project/[id]`)에 대한 UI 명세입니다.

## 1. 레이아웃 구조 (Layout Structure)
- **상단 (Header)**: 메인 페이지와 동일한 네비게이션 바.
- **섹션 1: 프로젝트 헤더 (Project Header)**: 제목, 기간, 역할, 주요 링크 등 핵심 메타데이터.
- **섹션 2: 미디어 갤러리 (Media Gallery)**: 대형 이미지 슬라이더 또는 그리드.
- **섹션 3: 상세 설명 (Description)**: 프로젝트 배경, 문제 해결 과정, 성과 등 본문.
- **섹션 4: 기술 스택 (Tech Stack)**: 사용된 기술 아이콘/태그 목록.
- **섹션 5: 네비게이션 (Page Navigation)**: 이전 프로젝트 / 다음 프로젝트 링크.
- **공통 컴포넌트**: **AI 챗봇 위젯** (이 프로젝트의 컨텍스트를 가진 상태).
- **하단 (Footer)**: 메인 페이지와 동일.

---

## 2. 각 섹션별 상세 (Section Details)

### 2.1 프로젝트 헤더 (Project Header)
- **배치**: 페이지 최상단, 여백 충분히 확보.
- **구성**:
    - **카테고리 뱃지**: `App Development` 등 상위 분류.
    - **프로젝트 제목**: 매우 큰 폰트 (H1).
    - **한 줄 요약**: 프로젝트의 핵심 가치를 담은 리드 텍스트.
    - **메타 정보 그리드**:
        - **기간**: `2023.01 - 2023.03`
        - **역할**: `UI/UX Design & Frontend dev`
        - **기여도**: `100% (Solo project)`
    - **링크 버튼**: `Live Demo` (Primary), `GitHub Repo` (Secondary, 아이콘 포함).

### 2.2 미디어 갤러리 (Media Gallery)
- **구성**:
    - **메인 이미지**: 가장 중요한 대표 스크린샷 1장 (Full width, Rounded corners).
    - **이미지 슬라이더/그리드**: 추가 스크린샷 2~4장. 클릭 시 '라이트박스(Lightbox)' 형태로 확대 보기 가능.
- **캡션**: 이미지 하단에 상황을 설명하는 짧은 캡션 ("메인 대시보드 화면", "모바일 반응형 뷰" 등).

### 2.3 상세 설명 (Description)
- **구성**:
    - 텍스트 위주의 본문 영역 (Markdown 렌더링).
    - **구조 제안**:
        1.  **배경 (Background)**: 이 프로젝트를 왜 시작했는가?
        2.  **문제 (Problem)**: 어떤 어려움이 있었는가?
        3.  **해결 (Solution)**: 기술적으로 어떻게 해결했는가? (코드 스니펫 등 포함 가능)
        4.  **성과 (Result)**: 정량적/정성적 결과 (예: "로딩 속도 50% 개선").

### 2.4 기술 스택 (Tech Stack)
- **구성**:
    - **타이틀**: "Used Technologies"
    - **목록**: 아이콘 + 기술명 조합의 카드 또는 칩(Chip) 나열.
    - **분류**: Frontend / Backend / Deployment 등으로 나누어 보여주면 좋음.

### 2.5 하단 네비게이션 (Page Navigation)
- **구성**:
    - **이전 프로젝트**: `← [이전 글 제목]` (좌측 배치)
    - **목록으로**: `List Icon` (중앙 배치)
    - **다음 프로젝트**: `[다음 글 제목] →` (우측 배치)
- **스타일**: 호버 시 배경색 변경 및 화살표 애니메이션.

### 2.6 AI 챗봇 위젯 (Context Aware)
- **특징**: 상세 페이지 진입 시, AI의 시스템 프롬프트(Context)에 **현재 보고 있는 프로젝트의 정보**가 최우선으로 주입됨.
- **자동 트리거**: 사용자가 상세 설명을 50% 이상 읽었을 때(스크롤), "이 프로젝트의 기술적 구현 방식이 궁금하신가요?" 같은 맞춤형 팝업 메시지 출력.

## 3. 반응형 동작 (Responsive Behavior)

### 📱 모바일 (Mobile, < 768px)
- **헤더**: 제목 폰트 사이즈 축소. 메타 정보 그리드는 2열 또는 1열로 변경.
- **미디어**: 슬라이더(Swipe) 형태로 변경하여 터치 인터페이스 최적화.
- **네비게이션**: 이전/다음 버튼을 상하(Vertical) 배치하거나 단순화(화살표만 표시)하여 공간 확보.

### 💻 데스크톱 (Desktop, > 1024px)
- **레이아웃**: 상세 설명(본문)과 기술 스택(사이드바)을 좌우(2-Column)으로 배치하는 변형 레이아웃도 고려 가능. (기본은 1-Column 중앙 정렬).
- **이미지**: 고해상도 이미지 제공. 라이트박스 기능 활성화.
