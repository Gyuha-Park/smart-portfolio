# 2. 핵심 기능 명세 (Functional Specifications)

본 문서는 [PRD_Master_Index](./PRD_Master_Index.md)의 하위 문서입니다.

## 2.1 자료 자동 학습 (Zero-Config RAG)
| 항목 | 내용 |
| :--- | :--- |
| **기능 이름** | PDF/Link 업로드 및 자동 임베딩 |
| **설명** | 사용자가 자신의 포트폴리오(PDF 파일)나 Notion/Blog 링크를 입력하면, 시스템이 텍스트를 추출하고 벡터화(Embedding)하여 지식 베이스를 구축함. |
| **우선순위** | **필수 (P0)** |
| **사용자 시나리오** | 1. 김코드가 관리자 페이지에 로그인한다.<br>2. '자료 업로드' 버튼을 누르고 이력서 PDF를 드래그 앤 드롭한다.<br>3. 시스템이 "자료를 분석 중입니다..."를 표시하고, 30초 내에 "학습 완료" 메시지를 띄운다.<br>4. 이제 챗봇이 해당 PDF 내용을 기반으로 답변할 수 있게 된다. |

## 2.2 적극적 세일즈 AI 챗봇 (Active Sales AI)
| 항목 | 내용 |
| :--- | :--- |
| **기능 이름** | RAG 기반 컨텍스트 인식 챗봇 & 세일즈 트리거 |
| **설명** | 방문자의 질문에 대해 학습된 자료를 바탕으로 답변하며, 답변 후에는 자연스럽게 추가 제안이나 연락처 입력을 유도하는 꼬리 질문을 던짐.<br>**[예외 처리]** 포트폴리오와 무관한 질문(예: "오늘 날씨 어때?", "주식 추천해줘")에는 "죄송합니다. 저는 포트폴리오와 관련된 질문에만 답변드릴 수 있습니다."와 같은 정중한 거절 메시지를 출력하여 AI의 역할을 명확히 한정함. |
| **우선순위** | **필수 (P0)** |
| **사용자 시나리오** | 1. 박대표가 사이트에 접속하자 챗봇이 "안녕하세요, 김코드의 AI 비서입니다. 쇼핑몰 구축 경험이 궁금하신가요?"라고 먼저 말을 건다.<br>2. 박대표가 "네, 포트폴리오 좀 보여줘요"라고 하면, AI는 관련 프로젝트 요약과 함께 **관련된 '작품 상세 페이지' 링크 카드**를 띄워준다.<br>3. 박대표가 카드를 클릭하여 상세 내용을 확인한다.<br>4. (예외 상황) 박대표가 "주식 추천해줘"라고 묻자, AI는 정중히 거절한다. |

## 2.3 리드 수집 및 대시보드 (Lead Capture)
| 항목 | 내용 |
| :--- | :--- |
| **기능 이름** | 대화 내 연락처 추출 및 리드 관리 |
| **설명** | 채팅 로그에서 이메일, 전화번호, 의뢰 내용을 자동으로 추출하여 DB에 저장하고, 주인(사용자)에게 대시보드 형태로 보여줌. |
| **우선순위** | **필수 (P1)** |
| **사용자 시나리오** | 1. 김코드가 자고 일어난 후 대시보드를 확인한다.<br>2. '밤새 들어온 문의: 3건' 알림을 본다.<br>3. 박대표가 남긴 이메일과 "쇼핑몰 리뉴얼 견적 요청"이라는 요약 내용을 확인하고 바로 메일을 보낸다. |

## 2.4 페르소나 설정 (Custom Persona)
| 항목 | 내용 |
| :--- | :--- |
| **기능 이름** | 챗봇 톤앤매너 및 인사말 커스터마이징 |
| **설명** | AI의 말투(정중함/친근함), 호칭, 첫 인사 메시지, 주요 강조 포인트(기술력 강조/친절함 강조)를 설정할 수 있음. |
| **우선순위** | **필수 (P1)** |
| **사용자 시나리오** | 1. 디자이너 사용자는 "친근하고 감성적인" 톤을 선택한다.<br>2. 개발자 김코드는 "신뢰감 있고 전문적인" 톤을 선택하고, System Prompt에 "자신감 있게 답변할 것"을 추가한다. |

## 2.5 관리자 접근 제어 (Admin Access Control)
| 항목 | 내용 |
| :--- | :--- |
| **기능 이름** | 특정 이메일 기반 관리자 접속 제한 |
| **설명** | 관리자(Admin) 페이지는 아무나 접근할 수 없으며, 사전에 등록된 특정 이메일(Owner)로 로그인한 사용자만 접근 가능하도록 제한함. 일반 방문자는 오직 공개된 포트폴리오 페이지와 챗봇 인터페이스만 볼 수 있음. |
| **우선순위** | **필수 (P0)** |
| **사용자 시나리오** | 1. 'user@example.com' (소유자)이 로그인하면 관리자 대시보드로 이동함.<br>2. 로그인하지 않은 사용자나 허가되지 않은 계정이 `/admin` 페이지에 접근 시도 시, 자동으로 메인 페이지로 리다이렉트되거나 "접근 권한이 없습니다" 메시지 표시. |

## 2.6 프로젝트 목록 및 상세 (Project Display)
| 항목 | 내용 |
| :--- | :--- |
| **기능 이름** | 프로젝트 목록 뷰 및 상세 페이지 |
| **설명** | **[프로젝트 뷰]** 메인 화면에서 등록된 프로젝트들의 썸네일, 제목, 간단한 태그(기술 스택 등)를 그리드 형태로 보여줌.<br>**[상세 페이지]** 썸네일 클릭 시 이동하며, 프로젝트의 상세 설명, 이미지 슬라이더, 데모 링크, 사용 기술 등을 보여줌. AI 챗봇은 이 상세 페이지의 내용을 우선적으로 학습하고 언급함. |
| **우선순위** | **필수 (P0)** |
| **사용자 시나리오** | 1. 방문자가 사이트에 접속하면 챗봇과 함께 하단(또는 측면)에 'Project Section' 섹션을 본다.<br>2. 썸네일을 훑어보다 관심 가는 'E-commerce App' 프로젝트를 클릭한다.<br>3. 상세 페이지에서 스크린샷들을 넘겨보고 사용된 'Next.js', 'Supabase' 태그를 확인한다.<br>4. 궁금증이 생겨 우측 하단 챗봇에게 "이 프로젝트 개발 기간은 얼마나 걸렸나요?"라고 묻는다. |
