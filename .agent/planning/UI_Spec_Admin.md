# 🔐 관리자 페이지 (Admin) UI 명세서

이 문서는 Smart Portfolio의 관리자 기능에 대한 상세 UI 명세입니다.
인증된 사용자만 접근할 수 있는 영역입니다.

## 1. 관리자 로그인 (/admin/login)

### 1.1 개요
- **목적**: 관리자(소유자)의 접근 제어.
- **접근 권한**: Supabase Auth에 등록된 **특정 이메일(Owner)** 계정만 허용.

### 1.2 UI 구성
- **배경**: 심플한 단색 배경 또는 그라데이션 (중앙 집중형 카드 UI).
- **로그인 폼 (Login Form)**:
    - **타이틀**: "Admin Access Only".
    - **이메일 필드**: `<input type="email">` (필수).
    - **비밀번호 필드**: `<input type="password">` (필수).
    - **로그인 버튼**: `Sign In`.
- **에러 메시지**: "접근 권한이 없는 계정입니다." 또는 "비밀번호가 올바르지 않습니다."

---

## 2. 관리자 대시보드 레이아웃 (/admin)

### 2.1 공통 레이아웃 (Global Layout)
- **사이드바 (Sidebar)**: 좌측 고정 (너비 250px).
    - **로고**: 상단 "Smart Portfolio Admin".
    - **메뉴 리스트**:
        1.  `Dashboard` (홈)
        2.  `Profile` (프로필/스킬/경력 관리)
        3.  `Projects` (프로젝트 관리)
        4.  `Chatbot Data` (지식 베이스/RAG 관리)
        5.  `Leads` (문의 내역)
        6.  `Settings` (계정 설정)
    - **하단**: 로그아웃 버튼.
- **메인 컨텐츠 영역 (Content Area)**: 우측 나머지 영역 (반응형).

---

## 3. 메뉴별 상세 UI 명세

### 3.1 프로필 관리 (Profile Management)
- **URL**: `/admin/profile`
- **목적**: 메인 페이지의 '히어로', '스킬', '경력' 섹션 데이터 수정.
- **UI 구성**:
    - **기본 정보 폼**: 이름, 직군(Role), 한 줄 소개, 프로필 이미지 업로드.
    - **스킬 관리 (Skills)**:
        - `Frontend`, `Backend` 등 카테고리별 태그 입력 필드 (Tag Input).
        - 드래그 앤 드롭으로 순서 변경 가능.
    - **경력 관리 (Career)**:
        - **리스트**: 현재 등록된 경력 카드 목록.
        - **추가 버튼**: 클릭 시 '경력 추가 모달' 오픈.
        - **수정/삭제**: 각 아이템 우측 아이콘 버튼.

### 3.2 프로젝트 관리 (Project Management)
- **URL**: `/admin/projects`
- **목적**: 포트폴리오 프로젝트의 CRUD(생성/조회/수정/삭제).
- **UI 구성**:
    - **상단 액션 바**: `+ 새 프로젝트 추가` 버튼 (우측 상단).
    - **데이터 테이블 (Table)**:
        - **컬럼**: 썸네일(작게), 제목, 카테고리, 등록일, 상태(공개/비공개).
        - **액션**: 수정(Edit), 삭제(Delete) 버튼.
    - **추가/수정 모달 (Modal/Drawer)**:
        - **제목**: 텍스트 인풋.
        - **카테고리**: 셀렉트 박스 (Web, App, Design 등).
        - **대표 이미지**: 파일 업로더 (Preview 제공).
        - **상세 이미지**: 다중 파일 업로더.
        - **설명**: 마크다운 에디터 (WYSIWYG).
        - **관련 링크**: 데모 URL, 깃허브 URL 입력 필드.
        - **AI 학습 여부**: 체크박스 ("이 프로젝트 내용을 챗봇에게 학습시킴").

### 3.3 챗봇 데이터 관리 (Chatbot Data / Knowledge Base)
- **URL**: `/admin/chatbot-data`
- **목적**: RAG(검색 증강 생성)를 위한 지식 베이스 관리.
- **UI 구성**:
    - **업로드 영역 (Dropzone)**: "PDF 파일을 여기에 드래그하세요."
    - **학습 데이터 목록 (List)**:
        - **파일명**: `resume_2024.pdf`
        - **유형**: PDF / Text / Link
        - **상태**: `학습 완료` (초록색), `처리 중` (노란색), `실패` (빨간색).
        - **삭제 버튼**: 잘못된 데이터 삭제 시 벡터 DB에서도 제거됨.
    - **직접 입력 (Text Input)**:
        - 간단한 Q&A 세트나 특정 지시사항을 텍스트로 직접 추가하는 폼.

### 3.4 리드 관리 (Leads Management)
- **URL**: `/admin/leads`
- **목적**: 챗봇이 수집한 잠재 고객 연락처 확인.
- **UI 구성**:
    - **리스트 뷰**: 카드 형태 또는 테이블.
    - **정보**: 이름, 연락처, 문의 내용 요약, 수집 일시.
    - **상태 변경**: `확인 전` -> `연락 완료` 토글 버튼.
