# 📱 화면(Page) 목록 및 UI 설계 가이드

이 문서는 PRD(제품 요구사항 문서)를 기반으로 **Smart Portfolio** 서비스에 필요한 화면 목록을 정의합니다.

## 1. 🌐 공개 페이지 (Public Facing)

일반 방문자(채용 담당자, 잠재 클라이언트)가 접근하는 화면입니다.

### 1.1 메인 프로젝트 (Main Projects)
- **URL**: `/`
- **목적**: 프리랜서의 아이덴티티를 소개하고, 프로젝트 썸네일을 둘러볼 수 있는 메인 진입점.
- **관련 PRD 기능**:
    - `2.6` 프로젝트 목록 뷰 (필수)
    - `2.2` 적극적 세일즈 AI 챗봇 (플로팅 위젯 형태)
- **UI 설계 노트**:
    - **Hero Section**: 간단한 자기소개와 강렬한 카피 ("안녕하세요, 문제를 해결하는 개발자 김코드입니다").
    - **Project Grid**: 반응형 그리드 시스템 (모바일 1열, 데스크탑 3~4열).
    - **Chat Widget**: 우측 하단 고정(Sticky) 버튼. 클릭 시 채팅창 확장. 모바일에서는 화면 전체를 덮거나 하단 시트(Bottom Sheet) 형태 고려.

### 1.2 프로젝트 상세 (Project Detail)
- **URL**: `/project/[id]`
- **목적**: 특정 프로젝트의 상세 내용(이미지, 설명, 기술 스택, 링크)을 깊이 있게 보여줌.
- **관련 PRD 기능**:
    - `2.6` 프로젝트 상세 페이지 (필수)
    - `2.2` AI 챗봇 (컨텍스트 연동)
- **UI 설계 노트**:
    - **Image Slider**: 여러 장의 스크린샷을 넘겨볼 수 있는 캐러셀(Carousel).
    - **Info Block**: 기술 스택(Tags), 기간, 역할, 데모 링크 버튼.
    - **Context Trigger**: "이 프로젝트에 대해 AI에게 물어보기" 버튼을 두어, 클릭 시 챗봇이 해당 프로젝트 컨텍스트를 가지고 대화를 시작하도록 유도.

### 1.3 로그인 (Login)
- **URL**: `/login`
- **목적**: 관리자(소유자)가 대시보드에 접근하기 위한 인증 화면.
- **관련 PRD 기능**:
    - `2.5` 관리자 접근 제어
    - `3.3` 회원가입/로그인
- **UI 설계 노트**:
    - **Simple Form**: 이메일 입력 + 로그인 버튼 (Supabase Magic Link 또는 Password).
    - **Redirect**: 로그인 성공 시 `/admin`으로 이동. 일반 사용자는 접근 불가.

## 2. 🔐 관리자 페이지 (Admin Facing)

인증된 소유자만 접근 가능한 관리 화면입니다.

### 2.1 관리자 대시보드 (Admin Dashboard)
- **URL**: `/admin`
- **목적**: 현재 시스템 현황(리드 수, 학습 상태)을 한눈에 파악.
- **관련 PRD 기능**:
    - `2.3` 리드 수집 및 대시보드
    - `2.5` 관리자 접근 제어
- **UI 설계 노트**:
    - **Summary Cards**: '신규 문의', '총 방문자', '학습 문서 수' 등 핵심 지표 카드 배치.
    - **Recent Project**: 최근 등록한 프로젝트 리스트 간략 보기.

### 2.2 리드 관리 (Lead Management)
- **URL**: `/admin/leads`
- **목적**: AI 챗봇이 수집한 잠재 고객의 연락처와 문의 내용을 확인하고 관리.
- **관련 PRD 기능**:
    - `2.3` 리드 수집 및 대시보드
- **UI 설계 노트**:
    - **Data Table**: 날짜, 방문자 유형(추정), 문의 내용 요약, 연락처, 상태(답변대기/완료) 컬럼.
    - **Status Badge**: 'NEW', 'CONTACTED' 등으로 상태 구분.

### 2.3 지식 관리 (Knowledge Base / RAG)
- **URL**: `/admin/knowledge`
- **목적**: AI 학습을 위한 자료(PDF, 텍스트, 링크)를 업로드하고 관리.
- **관련 PRD 기능**:
    - `2.1` 자료 자동 학습 (Zero-Config RAG)
- **UI 설계 노트**:
    - **Drag & Drop Zone**: 파일 업로드 영역.
    - **Processing Indicator**: 학습 중/완료/실패 상태를 보여주는 리스트.
    - **Text Editor**: 필요 시 직접 텍스트를 입력해서 학습시킬 수 있는 간이 에디터.

### 2.4 프로젝트 관리 (Project Management)
- **URL**: `/admin/projects` (리스트), `/admin/projects/new` (등록)
- **목적**: 프로젝트 목록에 노출될 프로젝트 정보를 등록/수정/삭제.
- **관련 PRD 기능**:
    - `2.6` 프로젝트 목록 및 상세 (데이터 관리)
- **UI 설계 노트**:
    - **Image Uploader**: 대표 이미지(Thumbnail) 및 상세 이미지 다중 업로드.
    - **Rich Text Editor**: 프로젝트 상세 설명 작성 (Markdown 지원 권장).
    - **Tag Input**: 사용 기술(Stack)을 태그 형태로 입력.

### 2.5 설정 (Settings)
- **URL**: `/admin/settings`
- **목적**: AI 챗봇의 페르소나(말투, 인사말) 및 사이트 기본 정보 설정.
- **관련 PRD 기능**:
    - `2.4` 페르소나 설정
- **UI 설계 노트**:
    - **Form Inputs**: 봇 이름, 첫 인사 메시지, 시스템 프롬프트(고급 설정).
    - **Tone Selector**: 라디오 버튼이나 셀렉트 박스로 말투 선택 (정중하게/친근하게).
